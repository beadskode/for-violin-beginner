# 실행 검증 보고서 — 바이올린 1포지션 운지 안내

작성일: 2026-06-19

## 1. 실행 방법

서버 없는 정적 SPA(React + Vite + TypeScript). 로컬 개발 서버를 띄워 브라우저로 렌더를 확인했다.

```bash
npm install            # 의존성 설치
npm run dev            # Vite dev server (http://localhost:4318)
npm run build          # dist/ 정적 빌드 (GitHub Pages 배포용)
npm run preview        # 빌드 결과 미리보기
```

브라우저는 별도 헤드리스 도구(chromium-cli/playwright)가 없어, 설치된 **Google Chrome 헤드리스 모드**로 스크린샷을 캡처했다.

```bash
CHROME="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
# 데스크탑
"$CHROME" --headless --disable-gpu --hide-scrollbars \
  --window-size=1280,800 --screenshot=/tmp/violin_desktop.png http://localhost:4318/
# 모바일
"$CHROME" --headless --disable-gpu --hide-scrollbars \
  --window-size=390,844 --screenshot=/tmp/violin_mobile.png http://localhost:4318/
```

## 3. 화면 확인 내용

### 데스크탑 (좌: 조 선택 / 우: 지판 / 하단: 오선지)
- 좌측 조 선택: **샵(♯) 조 / 다장조·가단조 / 플랫(♭) 조** 3섹션으로 구분, 장·단조 모두 표시.
- 각 조 버튼에 **높은음자리표 + 조표를 그린 미니 오선지** SVG 정상 렌더(클렙 글리프 `𝄞` 정상 표시).
- 한글 조이름 크게 + 영어 작게 병기.
- 우측 지판: 1포지션 **모든 운지점 표시**. 선택 조(다장조) 스케일 음은 **선명**, 비스케일 음은 **흐리게**.
- 운지 스티커: **한글 음이름 크게 + 영어 작게**, 손가락 번호를 **색(0회색·1파랑·2초록·3주황·4빨강) + 숫자 배지** 병기.
- 하단 2행: 높은음자리표 오선지 정상(음 미선택 상태).

### 모바일 (세로 스택)
- 위에서부터 조 선택 → 지판 순으로 세로 스택 정상.
- 조 버튼 그리드가 초기엔 우측으로 살짝 넘쳐 3번째 열이 잘림 → **모바일 2열 고정**으로 수정 후 정상.

## 4. 발견·수정 사항
- (수정 완료) 모바일에서 조 버튼 그리드 오버플로 → `@media (max-width:720px)` 에 `.key-grid { grid-template-columns: repeat(2,1fr); }` 추가.

## 5. 남은 확인 필요 항목
1. **클릭 → 오선지 음표 표시 상호작용**: 정적 스크린샷으로는 클릭 전 상태만 확인. 로직은 구현·빌드 통과 상태이며, 실제 클릭 동작은 브라우저에서 직접 확인 권장.
2. **클렙 글리프**: 현재 macOS Chrome에서는 유니코드 `𝄞` 정상 렌더. 다른 폰트 환경(일부 Windows/Linux)에서 깨질 가능성 → 필요 시 SVG path로 교체.
3. **운지 손가락 매핑**: 반음 오프셋→손가락을 `[0,1,1,2,2,3,3,4]`로 단순화. 조별 엄밀 운지와 차이 가능.
4. **지판 비율/간격**: 800px대 화면에서 스크롤 없이 한 화면에 표시됨을 확인. 실제 바이올린 비율과의 근사 정도는 취향에 따라 추가 조정 가능.
